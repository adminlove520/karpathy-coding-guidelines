### Principles Included
1. **Think Before Coding** — Don't assume, surface tradeoffs
2. **Simplicity First** — Minimum viable code, nothing speculative
3. **Surgical Changes** — Touch only what you must
4. **Goal-Driven** — Define success criteria, loop until verified

---

## Why These 4 Principles?

| Principle | Fixes |
|-----------|-------|
| Think Before Coding | Silent assumption-making, hidden confusion |
| Simplicity First | Over-engineering, bloated abstractions |
| Surgical Changes | Side-effect edits, touching unrelated code |
| Goal-Driven | No clear finish line, weak success criteria |
